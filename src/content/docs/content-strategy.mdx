# Content Strategy

SEO は「技術設定」より「検索意図ごとの page 設計」が重要です。

## Content Pillars

| Pillar | Purpose | Example |
| --- | --- | --- |
| Product docs | 導入と仕様理解 | `superpowers-ja を Claude Code に install する` |
| Agent catalog | 長尾検索と比較 | `kintone developer agent` |
| Workflow tutorials | 実務 scenario | `Backlog ticket から受入条件を整理する` |
| Video pages | 検索 + 学習体験 | tutorial video + transcript |
| Blog | 方法論と開発ログ | `日本の SIer と AI agent workflow` |

## SEO Checklist

- 1 page = 1 search intent
- H1 は page topic をそのまま書く
- `description` を page ごとに設定
- Code blocks は copy しやすくする
- Video には transcript を付ける
- 関連 docs / tutorial / agent catalog へ内部 link する
- Agent と workflow は個別 page にする

## Publishing Policy

Public repository に置く content:

- docs
- tutorials
- video transcripts
- reusable examples
- product release notes

Private repository または別 storage に置く content:

- 未公開 launch plan
- 顧客名や案件情報
- analytics export
- 有料講座の完成 script
- sponsor / partner negotiation notes
