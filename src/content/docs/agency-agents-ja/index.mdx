## Two kinds of agents

### ⭐ Japan-market originals (97)

日本の現場向けに独自設計された agent。production-ready で、上流の英文 library には存在しない。

- **SIer / 受託開発 / 多重下請** workflow agents
- **kintone / LINE WORKS / Sansan / freee** integration specialists
- **個人情報保護法 / インボイス制度 / 電帳法** compliance agents
- **製造業 DX / 公共 sector** specialists

### Upstream-aligned skeleton agents (169)

上流 `msitarzewski/agency-agents` の agent inventory に 1:1 対応する agent。`upstream_path:` で元 file への mapping を保持。

現在は `translation_status: skeleton` の状態 — frontmatter、role boundaries、Japan-context wrapper は配置済みだが、本格的な prompt 翻訳は進行中。`grep` で進捗を確認できる:

```bash
grep -rl '^translation_status: skeleton'   . | wc -l   # 169
grep -rl '^translation_status: translated' . | wc -l   # 0
grep -rl '^translation_status: adapted'    . | wc -l   # 0
```

## Frontmatter schema

各 agent file は以下の frontmatter を持ちます:

| Field | Required | Values | Purpose |
|---|---|---|---|
| `name` | ✅ | string | agent 名 (日本語) |
| `description` | ✅ | string | 1 行 summary |
| `emoji` | ✅ | string | UI 用 emoji |
| `color` | ✅ | hex | UI 用 color |
| `source` | ✅ | `japan-original` \| `upstream` | 由来 |
| `upstream_path` | source=upstream 必須 | string | 上流 file への path |
| `upstream_name` | optional | string | 上流 agent の英文名 (日本語名が divergent な場合) |
| `translation_status` | optional | `skeleton` \| `translated` \| `adapted` | 翻訳状態 |

CI (`scripts/validate.sh`) でこの schema を強制しています。

## Main Use

```text
engineering/engineering-kintone-developer.md を使って、

```

## Workflow Runner

`superpowers-ja` の `workflow-runner` skill と組み合わせると、複数 role を順番に使う workflow を実行できます。

```text
superpowers-ja の workflow-runner を使って
agency-agents-ja/workflows/japanese-sier-requirements-review.yaml を実行してください。
```
